# QOIMA — Страница «Показатели»

## Что нужно сделать (3 шага)

### Шаг 1 — Удалить лишние папки из репозитория

Прошлые файлы были для Next.js, но ваш сайт — статический HTML + Vercel Functions.
Они не работают, просто лежат и не используются. Удалите:

- папку `app/` (целиком)
- папку `lib/` (целиком)
- файл `README.md` в корне (если хотите)

Это можно сделать прямо на GitHub: открыть папку → троеточие справа → Delete.

### Шаг 2 — Залить новые файлы

В этом архиве 2 файла:

```
api/pokazateli.js       ← Vercel Function (положить в существующую папку api/)
pokazateli.html         ← Страница (положить в корень рядом с index.html)
```

На GitHub: **Add file → Upload files** → перетащить файлы → Commit.

Главное чтобы пути были именно такие:
- `api/pokazateli.js` (внутри папки api/)
- `pokazateli.html` (в корне)

### Шаг 3 — Проверить

После того как Vercel задеплоит (около минуты), откройте:

**`https://qoima-one.vercel.app/pokazateli.html`**

Должна загрузиться страница «Показатели» с реальными данными из вашего Kaspi.

## Если не работает

**Если страница показывает «Ошибка загрузки»:**

1. Откройте `https://qoima-one.vercel.app/api/pokazateli` напрямую в браузере
2. Если видите ошибку про `KASPI_API_TOKEN не задан` — проверьте Vercel:
   - Settings → Environment Variables → должна быть переменная `KASPI_API_TOKEN`
   - После добавления нужно **передеплоить** проект (Deployments → ⋯ → Redeploy)
3. Если ошибка `Kaspi API 401` — токен неправильный или истёк, перевыпустите его в кабинете Kaspi

**Если 404 на странице:**
- Проверьте что файл лежит именно как `pokazateli.html` в корне репо
- Подождите 1-2 минуты пока Vercel задеплоит

## Как добавить ссылку «Показатели» в сайдбар вашего index.html

В файле `index.html` найдите вашу навигацию (там где «Дашборд», «Заказы Каспи» и т.д.)
и добавьте новый пункт, например первым:

```html
<a href="/pokazateli.html">Показатели</a>
```

Или можете заменить «Дашборд» на «Показатели» и поставить ту же ссылку.

## Что считается на странице

- **Сегодня / Неделя / Месяц** — количество выданных заказов, выручка, сравнение с прошлым периодом
- **График** — выручка по дням текущей и прошлой недели (две линии)
- **Просроченные** — заказы со статусом «принят», но не отправленные больше суток (Kaspi штрафует)
- **Возвраты** — заказы со статусом RETURNED

Данные тянутся напрямую из вашего Kaspi через `KASPI_API_TOKEN`. Демо-цифр нет — всё реальное.
