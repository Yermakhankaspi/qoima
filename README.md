# QOIMA — Чистая версия

Полностью пересобранный сайт с нуля. Простая структура без лишнего.

## Структура

```
api/
├── _lib/
│   ├── kaspi.js           ← обёртка Kaspi API (с чанками по 14 дней)
│   └── session.js         ← cookie-сессии
├── auth/
│   ├── login.js           ← POST /api/auth/login
│   ├── logout.js          ← POST /api/auth/logout
│   └── me.js              ← GET /api/auth/me
├── pokazateli.js          ← GET /api/pokazateli (аналитика)
├── orders.js              ← GET /api/orders (список)
├── order-status.js        ← POST /api/order-status (смена статуса)
└── kaspi-test.js          ← GET /api/kaspi-test (проверка связи)

index.html                 ← главная (Показатели, Заказы, Настройки)
login.html                 ← страница входа
package.json
```

## Установка на Vercel

### 1. Залить файлы на GitHub

Удалить из репозитория всё кроме `.git`, и залить все файлы из этого архива.

Самый простой способ — на GitHub:
- Открыть репозиторий
- Удалить ВСЕ существующие файлы и папки (`api/`, `index.html`, `login.html`, `package.json`, `INSTRUCTIONS.md`, `pokazateli.html`)
- Add file → Upload files → перетащить ВСЕ файлы и папки из распакованного архива
- Commit

### 2. Задать переменные окружения в Vercel

В Vercel → твой проект → Settings → Environment Variables.

Должны быть **три** переменные:

| Имя                 | Значение                                                         |
|---------------------|------------------------------------------------------------------|
| `KASPI_API_TOKEN`   | Твой токен из кабинета Kaspi (у тебя уже есть)                  |
| `ADMIN_PASSWORD`    | Пароль для входа в панель — придумай любой, минимум 8 символов  |
| `SESSION_SECRET`    | Случайная строка минимум 16 символов — например `qwerty123abc456` |

После добавления переменных нужно **передеплоить**:
Vercel → Deployments → последний → ⋯ → Redeploy.

### 3. Войти

Открыть `https://qoima-one.vercel.app/login.html` → ввести пароль (`ADMIN_PASSWORD`) → войти.

После входа откроется главная страница «Показатели».

## Что есть

- **Показатели** — продажи сегодня/неделя/месяц, сравнение, график, просроченные, возвраты
- **Заказы** — все заказы за 30 дней, по вкладкам (Новые, Моя доставка, и т.д.), клик → детали → смена статуса
- **Настройки** — тест связи с Kaspi

## Технические детали

- Чистый HTML/JS, без сборщиков, без React
- Vercel Functions (Node.js 18+) для бэкенда
- Авторизация — простая cookie с HMAC-подписью
- Лимит Kaspi 14 дней обходится автоматически (запросы идут чанками)

## Если что-то не работает

1. Открой `https://qoima-one.vercel.app/login.html` напрямую — если открылась форма входа, фронт работает
2. Войди и открой Настройки → «Проверить связь с Kaspi» — если ОК, всё подключено
3. Если ошибка — смотри текст ошибки, она подскажет какая переменная не задана

Если страница `/login.html` показывает 404 — значит файлы не залились правильно. Проверь что `login.html` лежит в **корне** репозитория.
