# QOIMA — Панель продавца Kaspi

Дашборд для продавцов Kaspi.kz с регистрацией пользователей и интеграцией Kaspi Partner API. Деплоится на Vercel.

- ✅ Защита API endpoints

## ⚠️ Что нужно сделать ОДИН РАЗ после обновления

Для регистрации нужна база данных. Vercel даёт её бесплатно:

### Создать Vercel KV (3 клика)

1. Открой свой проект на vercel.com
2. Сверху вкладка **Storage**
3. Нажми **Create Database** → выбери **KV** (Redis)
4. Имя любое (например `qoima-users`), регион **Frankfurt (fra1)** — ближе всего к Казахстану
5. Жми **Create**, потом **Connect to Project** → выбери Production, Preview, Development
6. Vercel сам добавит нужные переменные окружения (`KV_URL`, `KV_REST_API_URL`, `KV_REST_API_TOKEN`)

### Перезапустить проект

После того как KV подключена:
1. Deployments → у последнего деплоя → три точки → **Redeploy**
2. Готово. Открой сайт, увидишь страницу логина → нажми «Регистрация» → создай аккаунт.

## 🔐 Как это работает

- Пользователь регистрируется → пароль хешируется bcrypt → сохраняется в Vercel KV
- При входе создаётся сессия → её ID в httpOnly cookie на 30 дней
- При открытии сайта `index.html` спрашивает у `/api/auth/me`: «кто я?» Если не авторизован — редирект на `/login.html`
- API `/api/orders` тоже проверяет сессию перед запросом в Kaspi

## 🚀 Подключение Kaspi API

Отдельно от регистрации — для подключения реальных заказов из Kaspi нужен API-токен.

1. Получи токен у представителя Kaspi Shop
2. Vercel → Settings → Environment Variables → добавь `KASPI_API_TOKEN`
3. Redeploy

Документация Kaspi: https://guide.kaspi.kz/partner/ru/shop/api/general/q3193

## 💾 Лимиты Vercel KV (бесплатный тариф)

- 30 000 команд в день — этого хватит на сотни пользователей и тысячи входов
- 256 МБ хранилища
- При превышении сайт продолжит работать, но новые запросы могут отклоняться (платный тариф $1+)
